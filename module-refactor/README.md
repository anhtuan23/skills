# Module Refactor

This skill is for refactoring an existing code area into a structure that is
easier to navigate and easier to follow.

It pushes for a clean end state by:

- splitting long files by responsibility
- organizing code by feature ownership
- narrowing public surface
- removing stale compatibility code and dead helpers
- adding beginner-friendly comments to touched functions
- adding a local `README.md` when the feature needs a short overview
- refactoring tests to mirror the new production structure

Use it for requests such as:

- refactor this package
- split this long file
- clean up this module after migration
- make this code easier to navigate
